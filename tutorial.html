01
Primeiros Passos
Visão geral da interface e requisitos
🖥️ Visão Geral da Interface

O Biospeckle Tool Online é dividido em três áreas: o painel esquerdo com controles, seleção de algoritmo e parâmetros; a área central com a visualização do vídeo processado em tempo real; e o painel direito com histograma, estatísticas e informações do frame.

1
Requisitos do Sistema
Use um navegador moderno com suporte a WebRTC: Google Chrome, Firefox, Edge ou Opera GX. A câmera requer HTTPS — o GitHub Pages já oferece isso automaticamente.
2
Equipamento Necessário
Uma webcam ou câmera de smartphone. Para experimentos reais de biospeckle: ponteiro laser classe 2 (λ = 650 nm, P ≤ 5 mW) apontado para a amostra biológica, com a câmera capturando o padrão de speckle refletido.
3
Resolução de Vídeo
O app negocia automaticamente a maior resolução disponível da câmera — tentando 1920×1080, depois 1280×720 e por fim 640×480. A resolução negociada aparece na barra de status com o badge ● FHD, ● HD ou ● SD logo após ligar a câmera.
02
Controle de Câmera
Ativar, selecionar dispositivo, carregar vídeo e tela cheia
1
Ligar a Câmera
Clique em ⏻ LIGAR CÂMERA. O navegador solicitará permissão — clique em "Permitir". A câmera inicia, o vídeo ao vivo aparece na área central e a barra de status exibe a resolução negociada.
2
Selecionar Câmera
Se houver mais de uma câmera, o seletor CÂMERA aparece automaticamente abaixo do botão. Escolha o dispositivo desejado — a troca acontece sem precisar desligar e religar.
3
Carregar um Vídeo
Vá em Arquivo → Abrir Vídeo para processar um arquivo .mp4 ou .webm previamente gravado. O vídeo é reproduzido sem áudio e em loop. Todos os algoritmos e controles funcionam normalmente sobre ele.
⛶
Tela Cheia (Fullscreen)
Passe o mouse sobre a área de vídeo — um botão aparece no canto inferior direito da área de vídeo. Clique para expandir o canvas para tela cheia com fundo preto. Pressione Esc ou clique novamente no botão para sair. O processamento e a gravação continuam funcionando em fullscreen.
⚠️
Sobrecarga: Se o indicador ⚠ SOBRECARGA aparecer na barra de status, reduza o FPS Alvo ou desative o Filtro de Ruído. Câmeras de alta resolução + Gaussiano 7×7 com FPS alto podem saturar o processador.
03
Painel de Aquisição
FPS, Contraste, Filtro de Ruído e Kernel
📋 O que é o Painel de Aquisição?

O painel Aquisição controla como o frame bruto da câmera é preparado antes de ser enviado a qualquer algoritmo. Os ajustes de Contraste e Filtro de Ruído funcionam como um pré-processamento universal — afetam LASCA, MHI, Regiões e o modo Câmera igualmente.

⚡
FPS Alvo
Define quantos quadros por segundo o app processa (1 a 30). Entre 5 e 15 FPS é o intervalo ideal para a maioria dos experimentos de biospeckle — suficiente para capturar a dinâmica biológica sem sobrecarregar o processador.
🌗
Contraste
Slider de −100 a +100 (padrão: 0).

Valores positivos fazem esticamento linear do histograma: mapeia o intervalo mín–máx do frame para 0–255, realçando detalhes em imagens com pouco contraste (ambientes semi-escurecidos, câmeras de baixa sensibilidade).

Valores negativos comprimem o contraste em direção ao cinza médio (128), útil para reduzir saturação em câmeras muito sensíveis ou com exposição excessiva.

O ajuste é proporcional: +50 aplica metade do esticamento máximo, preservando mais a escala original.
🔧
Filtro de Ruído
Seletor com 4 opções de pré-processamento aplicado a cada frame antes dos algoritmos. O objetivo é reduzir o ruído eletrônico do sensor sem apagar o sinal real do padrão de speckle.
⬛
Tamanho do Kernel
Controla o raio do filtro escolhido: 1×1 (equivale a desativar), 3×3, 5×5 ou 7×7. Kernels maiores suavizam mais intensamente, mas aumentam o custo computacional — especialmente no filtro Mediana.
🔧 Tipos de Filtro de Ruído
Nenhum
Frame passa direto para o algoritmo sem nenhum tratamento. Use quando a câmera já produz imagem limpa ou para ver o sinal bruto de speckle.
Box
● Mais rápido
Média simples de todos os pixels na vizinhança. Implementado com tabela de área somada (SAT) — O(1) por pixel, independente do tamanho do kernel. Suavização uniforme, pode borrar bordas.
Gaussiano
● Melhor qualidade
Média ponderada com kernel separável — peso maior no centro, menor nas bordas. Suavização mais natural, preserva melhor as transições do padrão de speckle. Recomendado para LASCA.
Mediana
● Melhor preservação de bordas
Substitui cada pixel pelo valor mediano dos vizinhos. Excelente para ruído impulsivo (pixels isolados muito claros ou escuros). Preserva bordas melhor que Box e Gaussiano. Use com kernel 3×3 para manter velocidade.
💡
Recomendação prática: Para a maioria dos experimentos de biospeckle, comece com Gaussiano 3×3 + Contraste +20. Isso remove o ruído do sensor sem comprometer o padrão de interferência que os algoritmos precisam detectar. Para câmeras com muito ruído impulsivo, prefira Mediana 3×3.
✅
LASCA em Resolução Plena: O algoritmo LASCA agora usa tabela de área somada para calcular K = σ/⟨I⟩ em cada pixel individualmente (sem passo/step), mantendo a resolução total do vídeo. O indicador Resolução Plena ✓ no painel LASCA confirma este modo.
04
Algoritmos
Os quatro modos de processamento disponíveis
📷
Imagem da Câmera
SEM PROCESSAMENTO
Exibe o vídeo diretamente, sem algoritmo. Pode ser visualizado em modo Normal, Escala de Cinza ou com Coloração Artificial (pseudo-cores). Útil para verificar alinhamento da câmera e iluminação do laser antes de iniciar a análise.
🔬
LASCA
LASER SPECKLE CONTRAST
Implementa o Laser Speckle Contrast Analysis (Briers & Webster, 1996). Calcula o contraste K = σ/⟨I⟩ em janela deslizante pixel a pixel usando tabela de área somada. Regiões com maior atividade biológica têm menor contraste (K baixo → cores frias no mapa).
🌊
MHI
MOTION HISTORY IMAGE
Implementa o Motion History Image (Godinho et al., 2012). Acumula diferenças entre frames consecutivos em buffer temporal, revelando o histórico de movimento. Excelente para visualizar a dinâmica temporal de amostras biológicas.
🗺️
Regiões
DETECÇÃO DE ATIVIDADE
Detecta e destaca regiões de maior atividade. Fundo em escala de cinza, áreas ativas em vermelho. Usa detecção de gradiente (Sobel) com blur interno para eliminar o ruído do speckle antes da análise de borda.
✅
Boa prática: Comece com Imagem da Câmera para verificar o setup. Depois configure o Contraste e o Filtro de Ruído no painel Aquisição. Só então alterne para LASCA ou MHI para análise quantitativa.
05
Parâmetros dos Algoritmos
Como ajustar LASCA, MHI e Regiões
🔬 Parâmetros — LASCA
ParâmetroValoresDescrição
Resultado Colorido On / Off Quando ativo, aplica a paleta pseudo-cores (azul→ciano→verde→amarelo→vermelho). Desativado exibe em escala de cinza.
Janela (Máscara) 3, 5, 7, 9 Tamanho da janela deslizante para cálculo do contraste K = σ/⟨I⟩. Janelas maiores suavizam o mapa de atividade mas perdem detalhes finos. Padrão 9 é adequado para a maioria das amostras.
Resolução Plena ✓ Fixo O LASCA agora opera pixel a pixel com tabela de área somada — sem step/passo. A imagem de saída tem a mesma resolução do vídeo de entrada. Este indicador confirma o modo de alta resolução.
🌊 Parâmetros — MHI
ParâmetroValoresDescrição
Resultado Colorido On / Off Aplica a paleta pseudo-cores ao resultado do MHI, facilitando a visualização da intensidade de movimento.
Threshold 0 – 255 Limiar de sensibilidade para detecção de movimento entre frames. Valores baixos capturam movimentos sutis; valores altos ignoram variações leves. Recomendado: 20–60.
Buffer 2 – 32 Número de frames armazenados no histórico. Buffers maiores revelam movimentos mais lentos e acumulados ao longo do tempo.
🗺️ Parâmetros — Regiões
ParâmetroValoresDescrição
Threshold 0 – 255 Sensibilidade da detecção de borda (magnitude do gradiente). Valores menores detectam variações sutis; valores maiores destacam apenas as regiões de maior contraste.
Blur Radius 1 – 4 Raio do filtro Box Blur interno aplicado antes da detecção de borda. Raios maiores eliminam mais ruído de speckle, gerando regiões mais compactas e menos fragmentadas.
Espessura 1 – 5 Espessura visual do destaque vermelho nas regiões ativas. Não afeta o cálculo, apenas a visualização.
💡
Paleta de Cores: A paleta vai de azul (baixa atividade / baixo K) passando por verde até vermelho (alta atividade / alto K). Em LASCA, cores frias = mais atividade biológica (menor contraste de speckle). Em MHI, cores quentes = maior movimento.

Abrir Vídeo
Vá em Arquivo → Abrir Vídeo. Selecione um arquivo .mp4 ou .webm. A aba muda para VÍDEO e todos os algoritmos e controles de Aquisição continuam funcionando normalmente.

Dica de Workflow: Grave o experimento em vídeo primeiro. Depois carregue o arquivo, ajuste Contraste e Filtro de Ruído no painel Aquisição para obter a melhor imagem pré-processada, e salve o resultado completo com REC ou o frame atual com 💾 Salvar Imagem.
07
Salvar Resultado
Salvar imagem do frame ou gravar vídeo processado
💾
Salvar Imagem
Clique no botão 💾 Salvar Imagem na barra de abas (ao lado do botão REC). Salva o frame atual do canvas de processamento como .png. O nome do arquivo inclui o algoritmo ativo e um timestamp.
🎥 Como funciona a gravação de vídeo

O botão REC na barra de abas captura diretamente o canvas de processamento — exatamente o que aparece na tela, incluindo os efeitos de LASCA, MHI, Regiões, Contraste e Filtro de Ruído. O resultado é salvo como arquivo .webm.

1
Iniciar a Gravação
Com a câmera ligada, clique no botão REC (com tom avermelhado) na barra de abas. O botão muda para PARAR, o ponto vermelho começa a piscar e o cronômetro inicia a contagem 00:00, 00:01...
2
Parar e Salvar
Clique em PARAR. O arquivo é gerado automaticamente e o download inicia no navegador. O nome segue o padrão speckle_[algoritmo]_[timestamp].webm.
3
Parada Automática
Se a câmera for desligada durante a gravação, a gravação para automaticamente e o arquivo é salvo — sem perda dos dados já capturados.
💡
Indicador na aba: Enquanto a câmera está ativa a aba exibe AO VIVO. Ao carregar um arquivo de vídeo, exibe VÍDEO. Quando a câmera está desligada, a aba fica vazia.
⚠️
Codec: O arquivo é gravado em WebM (VP9 ou VP8 conforme o suporte do navegador). Para converter para MP4, use o VLC ou ffmpeg: ffmpeg -i arquivo.webm saída.mp4
08
Uso no Celular
Orientação, câmera traseira e resolução HD
📱
Use na Orientação Horizontal
No celular, o aplicativo deve ser usado na horizontal (paisagem). Em orientação vertical aparece um aviso pedindo para girar o dispositivo.
📷
Câmera Traseira em HD
Ao ligar a câmera no celular, o app ativa automaticamente a câmera traseira e negocia a maior resolução suportada (tentando 1920×1080 → 1280×720). A resolução real aparece na barra de status com o badge HD ou FHD.
⬇️
Usar Offline
Acesse Arquivo → Download do App (Offline) e baixe os arquivos do repositório GitHub. Abra index.html localmente — sem instalação necessária.
⚠️
Desempenho no celular: Com câmera em FHD e Filtro Gaussiano 7×7, o processamento pode ser intenso. Se aparecer ⚠ SOBRECARGA, reduza o FPS para 5–8 ou use kernel 3×3.
🌐
Navegador Recomendado para Celular: O app foi testado com Opera GX (Android/iOS). Se tiver problemas de câmera no Chrome ou Safari, experimente o Opera GX — disponível gratuitamente nas lojas de aplicativos.
09
Perguntas Frequentes
Dúvidas comuns e solução de problemas
A câmera liga mas a resolução mostrada é baixa (640×480). O que fazer?+
O app tenta 1920×1080 e 1280×720 antes de 640×480. Se ficar em SD, significa que o navegador ou o dispositivo não suportam resoluções maiores naquele contexto. Tente no Chrome mais recente, verifique se outra aba está usando a câmera com resolução bloqueada, ou teste em outro dispositivo.
Qual filtro devo usar para biospeckle?+
Gaussiano 3×3 é o ponto de partida recomendado — remove o ruído eletrônico do sensor sem borrar o padrão granular do speckle que os algoritmos precisam analisar. Se a câmera produz pixels isolados muito brilhantes ou escuros (ruído impulsivo), prefira Mediana 3×3. O filtro Box é mais rápido mas suaviza de forma menos uniforme.
O que faz o controle de Contraste no painel Aquisição?+
O contraste aplica um esticamento linear do histograma a cada frame. Valores positivos identificam o pixel mais escuro e o mais claro do frame e remapeiam esses extremos para 0 e 255 — aumentando a diferença visual entre regiões ativas e inativas. Isso é especialmente útil com câmeras de baixa sensibilidade ou em ambientes com pouca iluminação. Valores negativos comprimem o contraste em direção ao cinza médio, útil para câmeras muito sensíveis.
A câmera não aparece ou o navegador não pede permissão. O que fazer?+
Verifique se o site está em HTTPS (obrigatório). Se rodando localmente, use um servidor local (como o Live Server do VS Code) ou o GitHub Pages. Verifique também se outra aba ou programa está usando a câmera.
Qual a diferença entre LASCA e MHI?+
LASCA analisa o contraste espacial dentro de um único frame — mede a variação local de intensidade causada pelo padrão de speckle. MHI compara frames ao longo do tempo — acumula o histórico de diferenças temporais. LASCA é melhor para medir nível de atividade biológica pontual; MHI é melhor para visualizar a evolução e dinâmica do movimento ao longo do tempo.
Os dados são enviados para algum servidor?+
Não. Todo o processamento ocorre localmente no seu navegador via API Canvas do HTML5. Nenhuma imagem, vídeo ou dado é transmitido para servidores externos. A aplicação funciona completamente offline após o carregamento inicial da página.
O arquivo gravado pelo REC abre em qualquer player?+
O arquivo .webm abre diretamente no Chrome, Firefox, Edge e VLC. Para convertê-lo para MP4 (mais compatível com Windows Media Player, WhatsApp etc.), use o VLC (Converter/Salvar) ou o comando: ffmpeg -i arquivo.webm saída.mp4
Posso gravar enquanto analiso um vídeo carregado?+
Sim. O botão REC funciona tanto com a câmera ao vivo quanto com um arquivo de vídeo carregado via Arquivo → Abrir Vídeo. O que é gravado é sempre o canvas de processamento — o resultado visual com o algoritmo aplicado.
O botão de tela cheia funciona em celular?+
Sim, mas com uma ressalva: no celular o botão aparece normalmente, porém alguns navegadores móveis (especialmente Safari no iOS) restringem o fullscreen de elementos <canvas> individualmente. Se não funcionar, tente pelo Chrome para Android ou Opera GX, que oferecem suporte completo à API Fullscreen.
